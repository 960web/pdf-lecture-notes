# Changelog

## [1.1.0] - 2026-06-02

### 工作流精简

- Step 1.4 提问结构重组：C（详略风格）先行，A+B 合并为"笔记结构与元素处理方案"，提问从 7 类减为 6 类
- Step 1.5 notes-plan.md 模板优化，格式规范从 1 项扩展为 7 项完整清单

### 格式规则架构修复

- 格式规则从 OCR Agent 侧锚定到 Main AI 写作路径（notes-plan.md + Phase 3 Step 8）
- "禁止 `---` 分隔符"规则在写作契约中显式约束
- 树形图/结构图明确用嵌套无序列表，禁止代码块包裹（避免阻止 LaTeX 渲染）
- 新增空行规则：块级元素前必须加空行

### PDF 导出改进

- 新增 `--toc` 询问（Step 1.4 F），用户自主选择是否生成 PDF 目录
- 合并 PDF 改用 `cat` 按章节号显式排序，修复 glob 字典序乱序
- 新增 LaTeX 导言区文件 `pandoc-header.tex`（字体、数学、enumitem 紧排）
- Pandoc 命令统一加 `--include-in-header`，去掉不存在的 `-C` 参数

### 术语与规范优化

- "基础内容精讲" → "知识点梳理"
- 知识结构图明确要求含具体概念名称及层级关系
- Step 1.4 G 去重，交叉引用和页码标注合并到格式规范

## [1.0.1] - 2026-05-11

### 多渠道分发支持

- 新增 npm 包清单（`package.json`），发布至 npm registry
- 调整目录结构兼容 skills.sh / OSM / SkillsMP / SkillsGate 等注册中心
- 根目录保留 `SKILL.md` 入口，兼容要求根目录入口的注册中心

## [1.0.0] - 2026-05-11

### 初始版本

- 完整 4 阶段工作流：环境准备 → 方案制定 → Pilot 验证 → 批量制作
- 并行 OCR：后台 Agent + PaddleOCR-VL，~10 页/批次
- 通用方案文档：一份方案覆盖所有章节
- 用户交互节点：Phase 0/1/2/3/4 强制确认点
- 边界情况处理：
  - 跨页内容截断（1 页 overlap 机制）
  - PDF 无目录（文本密度/标题采样自动探测）
  - 混合 PDF（文字页直接读 + 图片页 OCR）
  - 中断恢复
  - PDF 加密
  - 页面旋转校正
  - 多栏/复杂排版警告
  - 文件名非法字符清理
  - 磁盘空间估算
  - API 配额预检
  - 数据隐私提醒
- 输出格式支持：Markdown / MD+PDF（整体/分章节）
- 失败重试机制：单批次重试 3 次
